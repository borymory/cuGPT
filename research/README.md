## **Work Log**

I chose to store all kernel and organizational related work here. Each markdown file contains personal diagrams/math over that specific layer kernel. This work log serves also as a timeline of cuGPT.

Implemented Language Model head and Sampler. LM Head uses weight tying, reutilizing the embedding weight matrix transposed to obtain logits from the final layernorm. Sampler choice is Online Softmax top-k fused (top k=50 by default), implemented from [this paper, algorithm 4](https://arxiv.org/abs/1805.02867) with some additional changes for removing redundant expf functions and excessive top-k checks calls. The resulting sampler takes in input logits[B, C] and outputs the next tokens[B] for each batch.

The following up task is setting up FlashAttn for the suitable tensor sizes, Q,K,V tensor preparation and causal mask application. Tests for fused layers will follow afterwards. Backward kernels are right now out of scope for complexity and accessibility reasons.

Extensive effort in Flash Attention forward kernel, utilizing Tensor Cores at FP16 for mixed precision. It is currently under development. The current focus is on proper inlined device helper functions to aid readable and clear data flow. For a single GPU to run inference locally, it needs to compute every 12 heads in a single run. This is achieved by using CUDA Grids and proper tensor striding to select batches and heads for parallel execution. A clear distinction in notation is to be made: d refers to d_head not d_model which is 768 in GPT-2. Particularly C = 768 and dividing the work between 12 heads gives us d_head = 768/12 = 64. Thus d = 64.

The direction in which the Flash Attention kernel development heading was misaligned with the cuGPT goals and is thus migrated to /dev. There will not be tensor operations for the time being in the kernels as element wise functions such as softmax require in-register access mapping to the wmma::fragment struct which varies significantly by GPU model, reducing accessibility. In its place will be a standard Flash Attention implementation utilizing standard matmul logic with mixed precision, proper tensor striding and inlined device helper functions.
